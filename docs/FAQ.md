# 常见问题

## 启动问题

### Q: 后端启动报错 "Address already in use"

**问题**：端口 8000 被占用

**解决方法**：
1. 查找占用端口的进程：
   ```bash
   netstat -ano | findstr :8000
   ```
2. 结束该进程：
   ```bash
   taskkill /F /PID <进程ID>
   ```
3. 或使用其他端口启动：
   ```bash
   python -m uvicorn main:app --port 8001
   ```

---

### Q: 前端启动报错 "Node version not supported"

**问题**：Node.js 版本过低

**解决方法**：
1. 检查 Node 版本：`node -v`
2. 使用 nvm 安装 18+：
   ```bash
   nvm install 18
   nvm use 18
   ```

---

### Q: npm install 失败

**问题**：依赖安装失败

**解决方法**：
1. 清除缓存：
   ```bash
   npm cache clean --force
   ```
2. 删除 node_modules 和 package-lock.json
3. 重新安装：
   ```bash
   npm install
   ```

---

## 工作流问题

### Q: 工作流不执行

**可能原因**：

1. **没有起始节点**：工作流必须有输入节点作为起点

2. **节点未连接**：检查节点之间的连线是否正确

3. **配置错误**：检查节点的必填配置项是否填写

**排查步骤**：
1. 确认有数据输入节点
2. 确认节点之间的连线正确（输出 → 输入）
3. 查看底部日志面板的错误信息

---

### Q: 节点添加后不显示

**问题**：从工具栏拖拽节点到画布后，节点没有出现

**解决方法**：
1. 刷新页面
2. 检查浏览器控制台是否有错误
3. 重启后端服务（重新加载插件列表）

---

### Q: 节点执行失败

**问题**：执行时节点报错

**解决方法**：
1. 查看日志面板的具体错误信息
2. 常见错误：
   - **文件路径错误**：检查配置中的文件路径是否存在
   - **数据类型不匹配**：确认输入输出数据类型一致
   - **配置值错误**：检查阈值等数值参数是否合理

---

## 日志问题

### Q: 日志面板不显示日志

**可能原因**：

1. **WebSocket 未连接**：刷新页面重试

2. **日志级别过滤**：确认选择的是"全部"或对应标签

**解决方法**：
1. 刷新页面
2. 检查浏览器控制台是否报错
3. 点击"清空"按钮后再试

---

### Q: 后端日志不显示

**问题**：只有前端日志，没有后端日志

**解决方法**：
1. 检查后端是否启动
2. 检查 WebSocket 连接是否正常
3. 查看后端控制台是否有日志输出

---

## 插件问题

### Q: 自定义插件不显示

**可能原因**：

1. **目录结构错误**：必须在 `backend/nodes/{分类}/{插件名}/` 下

2. **metadata.json 格式错误**：检查 JSON 语法

3. **main.py 有语法错误**：

**解决方法**：
1. 检查目录结构是否正确
2. 验证 metadata.json 格式（可用 JSON 校验工具）
3. 重启后端服务
4. 刷新前端页面

---

### Q: 插件执行报错 "No module named 'xxx'"

**问题**：插件使用了未安装的 Python 模块

**解决方法**：
1. 安装所需模块：
   ```bash
   pip install <模块名>
   ```
2. 或在插件目录创建虚拟环境

---

## 数据问题

### Q: 节点之间的数据传递失败

**可能原因**：

1. **端口不匹配**：连接的端口类型不一致

2. **数据格式错误**：输出 JSON，输入却期望其他格式

**解决方法**：
1. 检查连接线两端的端口类型
2. 查看日志中数据格式
3. 使用"数据日志"节点查看中间数据

---

### Q: 如何查看工作流执行结果

**方法**：
1. 执行完成后，结果会在日志面板显示
2. 也可在输出节点配置中指定输出路径
3. 查看 `backend/workflows/` 目录下的输出文件

---

## 其他问题

### Q: 如何备份工作流

**方法**：
1. 复制 `backend/workflows/{名称}.workflow` 文件
2. 文件是纯 JSON，可直接查看和编辑

---

### Q: 如何重置工作流画布

**方法**：
1. 点击顶部工具栏的"新建"按钮
2. 或直接刷新页面（未保存的工作流会丢失）

---

### Q: 浏览器控制台报错 "CORS"

**问题**：跨域请求被阻止

**解决方法**：
1. 确认后端已启动
2. 检查后端 CORS 配置是否正确
3. 确认前端访问地址在允许列表中

---
